# AI Agent 项目环境配置说明

## 环境基本信息

### 环境路径
- **虚拟环境位置**：`D:\BaiduNetdiskDownload\ai_agent_with_langchain\.venv`
- **Python 版本**：3.12.4
- **pip 版本**：24.2

### 环境建立方式
1. 项目使用 Python 3.12.4 创建虚拟环境
2. 通过 `pyproject.toml` 管理依赖（已设置 Python 版本要求为 `>=3.10`）
3. 使用 `uv` 和 `pip` 作为包管理工具

## 依赖管理

### 已生成的文件
- **pyproject.toml**：项目配置和依赖声明
- **uv.lock**：uv 包管理器的依赖锁定文件
- **requirements.txt**：已导出的依赖列表（适用于 pip 安装）

### 安装其他库的方法

#### 方法一：使用 uv 安装（推荐）
```bash
# 确保已激活虚拟环境
.venv\Scripts\Activate.ps1  # PowerShell
# 或
.venv\Scripts\activate.bat  # Command Prompt

# 安装单个库
uv pip install <库名>

# 安装特定版本
uv pip install <库名>==<版本号>

# 示例
uv pip install pandas==2.2.0
```

#### 方法二：使用 pip 安装
```bash
# 确保已激活虚拟环境
pip install <库名>

# 从 requirements.txt 安装所有依赖
pip install -r requirements.txt
```

## 环境使用指南

### 激活虚拟环境
- **PowerShell**：
  ```bash
  .venv\Scripts\Activate.ps1
  ```
- **Command Prompt**：
  ```bash
  .venv\Scripts\activate.bat
  ```

### 退出虚拟环境
```bash
deactivate
```

### 查看已安装的库
```bash
# 查看所有已安装的库
pip list

# 查看特定库
pip list | findstr "<库名>"  # Windows
```

### 更新依赖
```bash
# 更新特定库
uv pip install --upgrade <库名>

# 或使用 pip
pip install --upgrade <库名>
```

### 导出依赖到 requirements.txt
```bash
pip freeze > requirements.txt
```

## 注意事项

1. **环境一致性**：在团队协作中，建议始终使用 `requirements.txt` 或 `uv.lock` 来确保环境一致性
2. **版本兼容性**：安装新库时注意版本兼容性，特别是与 langchain 相关的库
3. **代理设置**：如需通过代理安装，可配置 pip 或 uv 的代理设置
4. **清理环境**：如需重建环境，确保完全删除旧的 `.venv` 目录后再创建新环境
